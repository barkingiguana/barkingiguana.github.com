--- title: "Git: calculating the short sha1 hash of a commit" layout: post author: Craig ---

I frequently find myself referring to commits in a repository managed by Git. Until tonight I didn't know how to get the short version of the commit hash so I had to refer to the full thing. f328b61397cd8a7dfddb005d4d0a3280a0777a46. Yuck.

To get the short version use git rev-parse --short --verify [commit-id].

git rev-parse --short --verify f328b61397cd8a7dfddb005d4d0a3280a0777a46
f328b61

If you'd like to transform that back to the long commit hash again just drop the --short option.

git rev-parse --verify f328b61
f328b61397cd8a7dfddb005d4d0a3280a0777a46
